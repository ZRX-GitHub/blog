自动类型转换

类型向下不兼容，如long 转 int，精度会丢失，只运行向上兼容。

char、short、byte进行运行时，比int小的，都会自动默认转换为int，

float、int，自动转换float。

double > float > long > int > short 、char、byte

short a = 1；
byte b = 2；
short c = a + b;  //编译时会提示，int转short会有损失

byte d1 = 1;
byte d2 = 2;
byte d3 = 1+2;    //正确
byte d4 = d1+d2;  //错误: 不兼容的类型: 从int转换到byte可能会有损失

final int NUM = 2; // 把变量定义为最终变量，即NUM=2变为常量，值不再改变。

 

强制转换

PS：会造成精度丢失

int a = 1;
byte b = (byte)a;  // 1

int a = 128;
byte b = (byte)a;  // -128

逻辑运算符

&（与）、|（或）、！（非）、&&（短路与）、||（短路或）、^（异或）。

文字表达：

&（与） 与 &&（短路与）的区别：

&：不管前面的条件是bai否正确，后面都执行；

&&：前面条件正确时，才执行后面，不正确时，就不执行，就效率而言，这个更好。

 

三元运算符

判断式？算式1：算式2

int a=2, b=3, c;

c = a>b ? 4:5;  //c=5

即a>b不成立，取算式2。

建议：算式1：算式2的类型一致。

 

位运算符

&（按位 与）、|（按位 或）、~（按位 非）、^（按位 异或）、>>（有符号右移）、<<（有符号左移）、>>>（无符号右移）、>>= （右移赋值）、>>>= （右移赋值，左边空出的位以0填充）、<<= （左移赋值）。

链接：原码、反码、补码三者之间的关系。

二进制 转 十进制

// &（同位都为1，才为1）
0101 & 0111 = 0101

// |（存在位为1，就为1）
0101 | 0111 = 0111

// ~（存在位为1，就为0）
~0101 = 1010

// ^（同位相同为0，不同为1）
0101 ^ 0111 = 0010

// >>（有符号右移，空位使用0来补位，符号位不动）
0101>>2 = 1101
1101>>2 = 1111

// >>（有符号左移，空位使用0来补位，符号位不动）
0101<<2 = 0111 // 变成负数了
1101<<2 = 10100

// 即若该数为正，则高位补0，而若该数为负数，则右移后高位同样补0。
// -10 = 1111 1111 1111 1111 1111 1111 1111 0110 
1111 1111 1111 1111 1111 1111 1111 0110>>>2 
= 0011 1111 1111 1111 1111 1111 1111 1101

// >>= （右移赋值）

PS：非二进制也是一样可以的，使用二进制只是为了更好观察；

负数在计算机内部是以补码形式存放的：如果数字是一个无符号数值，则用0填补最左边的N位， 如果数字是一个有符号数值，则用1填补最左边的N位。
