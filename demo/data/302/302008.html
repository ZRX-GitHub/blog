封装类

为什么要使用封装（即作用、优点），封装的意义是什么？

优点：

1.提高代码的安全性（外部无法更改ATest1里面定义好的数据，注意，是定义好的）；

2.提高代码的复用性（可重复调用）；

3.“高内聚”：封装细节，便于修改内部代码(比如ATest1类里面定义的私有属性可以改为int啊什么其它类型的)，提高可维护性；

4.“低耦合”：简化外部调用，便于调用者使用，便于扩展和写作。

 

实例：

package Test1;

public class Test1 {
    public static void main(String[] args) {
        ATest1 at = new ATest1();
        // set方法用来定义数据
        at.setName("小明");
        at.setAge(11);

        at.show();

    }
}

package Test1;

public class ATest1 {
    // private（私有的）也是封装的一个体现。被private修饰的成员变量只能在 本类 中使用，外部不能直接通过对象属性改变。
    private String name;  //  定义的类型——String类型
    private int age;   //  定义的类型——int类型

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        this.age = age;
    }

    // 用来输入（静态方法调用非静态方法）
    // get 方法用来获取数据
    public void show() {
        System.out.println("他的名字是：" + this.getName() + "，他今年" + this.getAge() + "岁~！");
    }
}

方法、继承都是封装的一种体现，private（私有的）也是封装的一个体现。被private修饰的成员变量只能在本类中使用，外部不能直接通过对象属性改变。获取和改变需要提供设置器（访问接口|访问方法）和访问器来进行操作。

 

 

备用思考：

为什么要用封装：有些属性不能直接赋值（例如，生活中不符合要求的），要保证数据的正确性；

怎么用封装：1、私有化成员变量 private 2、使用setter、getter（规范化），如 name的格式 setName()、getName

在我们的setter、getter中加入限制条件：保证数据的完整性和正确性

封装什么时候使用：书写一个javabean，必须将它的成员变量进行封装
